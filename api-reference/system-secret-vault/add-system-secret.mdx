---
title: 'Add System Secret'
openapi: 'POST /v1/api/add_system_secret/'
---

Add a new secret to the System Secret Vault for use in webhook templates.

## Base URL

All API requests should be made to:

```
https://api.hookpulse.io
```

## Example request

<RequestExample>

```bash cURL
curl -X POST https://api.hookpulse.io/v1/api/add_system_secret/ \
  -H "x-hookpulse-api-key: {{x-hookpulse-api-key}}" \
  -H "x-brand-uuid: {{x-brand-uuid}}" \
  -H "Content-Type: application/json" \
  -d '{
    "secret_key": "{{secret_key}}",
    "secret_value": "{{secret_value}}"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.hookpulse.io/v1/api/add_system_secret/', {

    secret_key: '{{secret_key}}',

const data = await response.json();

```

```python Python
import requests

    'x-hookpulse-api-key': '{{x-hookpulse-api-key}}',
    'x-brand-uuid': '{{x-brand-uuid}}',
    'Content-Type': 'application/json'

    'secret_key': '{{secret_key}}',
    'secret_value': '{{secret_value}}'
}

response = requests.post(url, headers=headers, json=payload)
print(response.json())
```


```php PHP
<?php

$url = 'https://api.hookpulse.io/v1/api/add_system_secret/';
$data = [
    'secret_key' => '{{secret_key}}',
    'secret_value' => '{{secret_value}}'
];

$ch = curl_init($url);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_POST, true);

curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'x-hookpulse-api-key: {{x-hookpulse-api-key}}',

    'Content-Type: application/json'
]);

$response = curl_exec($ch);
curl_close($ch);

echo $response;
?>
```

</RequestExample>

## Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `secret_key` | string | Yes | Unique identifier/key name for the secret |
| `secret_value` | string | Yes | The secret value to store |

## Example response

<ResponseExample>

```json
{
  "success": true,
  "details": "Variable added successfully in vault"
}
```

</ResponseExample>

## Error response

If a secret with the same key already exists:

<ResponseExample>

```json
{
  "error": "Key with same name already in database",
  "success": false,
  "secret_uuid": "{{secret_uuid}}"
}
```

</ResponseExample>

## Response fields

| Field | Type | Description |
|-------|------|-------------|
| `success` | boolean | Indicates if the request was successful |
| `details` | string | Success message (only present on success) |
| `error` | string | Error message (only present on error) |
| `secret_uuid` | string | UUID of the existing secret (only present on error) |
