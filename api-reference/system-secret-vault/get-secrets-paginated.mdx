---
title: 'Get Secrets Paginated'
openapi: 'GET /v1/api/get_secrets_paginated/'
---

Retrieve a paginated list of all system secrets stored in your vault.

## Base URL

All API requests should be made to:

```
https://api.hookpulse.io
```

## Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `page` | integer | Yes | Page number to retrieve (starts from 1) |

## Example response

```json
{
  "data": [
    {
      "key": "{{key}}",

      "system_secret_uuid": "{{system_secret_uuid}}"
    }
  ],
  "meta": {
    "page": {{page}},
    "total_count": {{total_count}},
    "per_page": {{per_page}},
    "total_pages": {{total_pages}},
    "has_next": {{has_next}},
    "has_prev": {{has_prev}}
  },
  "success": true
}
```

</ResponseExample>

## Response fields

### Data array

| Field | Type | Description |
|-------|------|-------------|
| `key` | string | The secret key name |
| `inserted_at` | string | ISO 8601 timestamp when the secret was created |
| `call_key` | string | Template syntax to use this secret in webhooks (e.g., `{{ ##key }}`) |
| `system_secret_uuid` | string | Unique identifier for the secret |

### Meta object

| Field | Type | Description |
|-------|------|-------------|
| `page` | integer | Current page number |
| `total_count` | integer | Total number of secrets |
| `per_page` | integer | Number of items per page |
| `total_pages` | integer | Total number of pages |
| `has_next` | boolean | Whether there is a next page |
| `has_prev` | boolean | Whether there is a previous page |

### Root fields

| Field | Type | Description |
|-------|------|-------------|
| `success` | boolean | Indicates if the request was successful |
| `data` | array | Array of secret objects |
| `meta` | object | Pagination metadata |
